# Model Selection Guide for Abridge

This document explains **how to choose an LLM model** for use with Abridge.

Abridge is an **editorial condensation engine**, not a creative writing system.
Model choice directly affects narrative fidelity, determinism, and trustworthiness.

This guide defines:
- What the model is required to do
- What characteristics matter most
- How to evaluate candidate models
- Common failure modes to avoid

---

## 1. What Abridge Requires From a Model

Abridge tasks the model with acting as:

> **A disciplined literary editor performing loss-averse condensation over long narrative text**

This is fundamentally different from:
- generic summarization
- creative rewriting
- opinionated analysis

The model must preserve story mechanics, not reinterpret them.

---

## 2. Core Model Requirements (In Priority Order)

When selecting a model, prioritize the following attributes **in this order**:

### 2.1 Narrative Fidelity (Critical)

The model must:
- Preserve all events, actions, decisions, and outcomes
- Preserve chronology and causal relationships
- Avoid merging or abstracting distinct events

If fidelity fails, the output is unusable.

- guardrail violations
- manual inspection overhead

are effectively more expensive.

## 3. What Abridge Does *Not* Need From a Model

Do not optimize for:

- Creativity
- Stylistic flair
- Emotional intelligence
- Literary interpretation
- Genre awareness

These traits actively increase risk.

Use **real pipeline data**, not synthetic benchmarks.

Recommended tests:

2. Condense a morally charged scene
3. Condense a dense plot chapter
4. Compare compression ratios across stages
5. Inspect guardrail violations
6. Compare cost per successful run
